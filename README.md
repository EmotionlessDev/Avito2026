## Генерация слотов

### Выбранный подход: Lazy generation
Слоты генерируются **в момент запроса** пользователем доступных слотов на конкретную дату, а не заранее.

### Как это работает:
1. Пользователь запрашивает слоты на 2026-04-01
2. Сервис проверяет: слоты уже существуют в БД?
3. Если да, то просто возвращает существующие слоты
4. Если нет, то генерирует слоты на эту дату, после чего возвращает их

### Почему этот подход лучше для данного проекта
1. **Экономия места в БД**
   - Не храним слоты на даты, которые никто не запросит
   - С ленивой генерацией: только реально запрошенные даты

2. **Пользователь всегда делает GET перед бронью**
   - Невозможно забронировать слот без предварительного получения списка
   - Значит первый запрос всё равно произойдёт → генерация оправдана
